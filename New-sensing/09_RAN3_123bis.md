## RAN3#123bis (2024.04.15–19, 长沙)

### 本次会议 SON/MDT 主要共识与增强内容

本次会议 R18 纠错并入第 9 章（9.1 LTE / 9.2 NR / 9.3 Rapporteur Corrections），SON/MDT 无专属小节，且大部分 SON 相关提案未被处理（time 限制，顺延至 RAN3#124）。实质性 agreed/endorsed 结论很少。

## 实际处理的条目
- MDT/Trace 相关：修正 TS 38.401 中 user consent for trace reporting 描述（CR0384，CMCC/Huawei），澄清 trace 可由 AMF 或 UDM 触发，并修正 "Signalling based MDT activation triggered by AMF involving E1 and F1" 的信令流程图描述；Rev R3-242176 endorsed（在 #124 转正 agreed）。
- NR-U SON：Nokia 提出多 target 节点场景下 DL LBT reporting 触发问题（R3-241558），多数公司（E///、QC、HW 等）不认可该问题（R18 已同意不支持此类 case），noted，无规范变更。
- LS 方面：RAN2 的 Reply LS on SPR（R3-241516）noted；上次会议输出的 Reply LS on user consent for trace reporting（R3-241115）列入 left over/pending actions。

## 未处理、顺延至 #124 的 SON 提案（仅列入议程，无结论）
- Addition of SON for fast MCG recovery in stage 2、Addition of CPAC detection in stage 2（draftCR）。
- Inter-system MRO 系列纠错（TS 38.300/36.300 R16/R17/R18 draftCR，Samsung/ZTE/Nokia/CMCC）。
- PSCell List of RA report 的 stage2（TS 38.401 CR0387）与 X2AP/XnAP（CR1788/CR1270）纠错。
- Stage 2 correction for NR-U（TS 38.401 CR0388）、SN-initiated flag signalling for SPR（讨论稿）、Correction to 38.423 for SON features enhancement（CR1268）。

## 结论
本次会议除 trace reporting user consent 的 stage2 澄清（endorsed）外，无其他 R18 SON/MDT 相关 agreed 结论；主要 SON/MDT 纠错（fast MCG recovery、CPAC detection、inter-system MRO、RA report PSCell List、NR-U stage2、SPR）均顺延至 RAN3#124 处理。

### 本次会议 AI/ML 主要共识与增强内容

R18 AI/ML 纠错分散在第 9 章（CB # 7_R18AIRAN，moderator NEC）。注意：议程 10/11 的 multi-hop UE trajectory、AI/ML based ES 等为 Rel-19 SI 内容，不属于本摘要范围。

## 已达成的纠错（endorsed）
- UE trajectory 文本描述纠错（TS 38.423 CR1262，ZTE 等 → R3-242178 endorsed）：澄清 Measured UE Trajectory IE 指 UE 切换到目标节点后实际连接的 NG-RAN cells；Predicted Trajectory Cell Information IE 指 cell based UE trajectory prediction 的预测 cell 信息；Cell Based UE Trajectory Prediction IE 指预测 UE 将连接的 NG-RAN cells——统一并纠正三个 IE 的语义描述。
- Stage 3 corrections for AI/ML for NG-RAN（TS 38.423 CR1267，Huawei/Nokia/QC/InterDigital → R3-242235 endorsed），并增补 E///、CATT 联署（在 #124 转正 agreed）。

## 讨论未决、留待后续的议题
- OAM requirements for AI/ML stage 2（NEC/Huawei draftCR → R3-242180 noted）：Nokia 坚持等 SA5 结论（依赖 mapping rule），会议决定结合 SA5 进展再查。
- Stage 2 纠错中 "the details of model performance feedback are also out of 3GPP scope" 的表述被 DT/TI 质疑（RAN3 从未定义 model performance feedback），需要修正，draftCR（R3-242215）noted。
- 是否在规范中引入 AI/ML Model Training 与 AI/ML Model Inference 术语（terminology）：CB 结论为 "To be continued"，顺延至 #124（最终在 #124 以引用 SA5 TS 28.105 定义方式解决并 LS SA5）。
- Correction on Data Collection procedures over Xn（CMCC 等，CR1264）未获处理，顺延。

## 小结
本次会议 agreed/endorsed 的 R18 AI/ML 结论有限：仅 UE trajectory 相关 IE 语义澄清与一份 stage 3 综合纠错 CR 获 endorsed；AI/ML 术语定义、model performance feedback 表述、OAM requirements 等均未决，顺延至 RAN3#124。
