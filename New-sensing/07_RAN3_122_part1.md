## RAN3#122 (2023.11.13–17, 芝加哥；R18 两 WI 收官)

### 本次会议 SON/MDT 主要共识与增强内容

本次会议宣布 R18 SONMDT WI 在 RAN3 完成，全部 BLCR（38.401/38.300/36.300/38.413/38.423/36.423/38.470/38.420/38.473/37.340/37.320）endorse 定稿。以下仅为 #122 新达成的结论。

**SHR/SPR**
- 最终敲定：Target C-RNTI 同时加入 Xn 和 NG 的 HANDOVER REPORT 消息，用于 inter-RAT SHR 与 LTE RLF Report 的关联（correlation）。
- HO Report 消息（Xn 与 NG）中加入 timeSinceFailure IE（由 TimeSinceFailure-r11 更名而来），对应 TP（38.413/38.423）获批。
- SPR 收尾：在 S-NODE MODIFICATION REQUIRED 消息中新增 "SPR availability in UE" IE，SN 借此告知 MN UE 已生成 SPR，MN 可据此取回报告（38.423 TP 获批）。
- 关闭遗留问题：source SN 向 MN 传递 preferred T310/T312 SPR thresholds 或 timer values 作为辅助信息——R18 不做（讨论关闭）。
- 38.300 SHR stage2 TP、38.413 inter-RAT SHR/SPR TP 均获批，特性定稿。

**MRO**
- 本次以 stage2/stage3 清理收尾：CPAC SON 增强 TP（37.340，Samsung）、MRO TP（36.300，Huawei）、38.413 MRO TP（Ericsson）获批。
- Voice fallback MRO 的 38.300 stage2 清理 TP（ZTE 等）获批。
- 遗留开放点（suitable PSCell ID 从发起节点传给 target SN、fast MCG recovery 中 last serving MN 向 SN 转发 SCG 失败 RLF report）讨论后未达成共识，R18 不引入。

**RACH Enhancements**
- 38.423 与 36.423 的 RACH Enhancement 清理 TP（Huawei 等）获批，RACH INDICATION 相关规范文本定稿（每消息最多 64 个 RA Report Indication 等此前结论落入 CR）。
- 收到 RAN2 回复 LS：UE based 方案相关标准化工作由 RAN2 收尾，RAN3 跟随 RAN2 决定，不再另行开工。

**NPN**
- 38.423 MDT support in NPN TP 获批：保留异常情况（abnormal condition）描述——当 Area Scope for MDT IE 缺失而 PNI-NPN Area Scope 存在时的处理，关闭 #121bis 的 FFS。
- Ericsson 提议从 38.413/38.423 的 MDT Area Scope Choice 中删除 SNPN TAI Based MDT 与 SNPN Cell Based MDT——无共识，两个 area scope 保留在 R18 规范中。
- 38.413 的对应 TP 仅 noted，NG 口不再改动。

**SON for NR-U**
- 38.300 SON for NR-U stage2 TP 与 LBT failures in MRO TP 均获批。
- #121bis 的 WA 落地为方案（38.423 TP 获批）：在移动性失败场景，当 UE 重建回 source node 且 target node 能识别该 UE 时，支持上报切换执行期间 target 侧发生的 DL LBT failures；source node 请求时 target 提供 DL LBT 失败次数（新增请求指示 IE）。
- 放弃项：按 NR-U channel 新增"一致性 UL LBT 失败次数"负载度量（F1/Xn）——多数公司认为现有度量足够，R18 不引入。

**MDT Enhancements**
- 本次无实质新共识（cross-RAT logged MDT override 等此前已结论等待 RAN2，无新文稿处理）。

### 本次会议 AI/ML 主要共识与增强内容

本次会议宣布 R18 AI/ML for NG-RAN WI 在 RAN3 完成，38.401/38.300/38.420/38.423 四个 BLCR 全部 endorse 定稿。以下仅为 #122 新达成的结论。

**Stage2（12.2.1）**
- 38.300 stage2 描述更新 TP 获批（合并 Huawei/Lenovo 修改）：明确 "Support of AI/ML for NG-RAN, as a RAN internal function, is used to facilitate AI/ML techniques in NG-RAN"，并按在线评论删去对 legacy resource status reporting 的重复描述。
- 38.420 术语更正 TP 获批。
- 38.401 关于 E1/F1 接口 stage2 影响的 TP 明确 not pursued（呼应 split 架构 R18 不做的结论）。
- Ericsson 的"AI/ML 信息交换程序特征"描述与 Nokia 的接收节点行为 stage2 描述均仅 noted，不写入规范。

**LB（12.2.2.1）**
- Measurement Collection Duration IE 定为 INTEGER (1..5000, ...)，单位毫秒。
- "Average Packet Loss" IE 更名为 "Average Packet Loss DL"；Average Packet Delay IE 内新增 Average Packet Delay UL/DL 两个子 IE，取值 INTEGER (0..10000)。
- Predicted Resource Status IE 最终只含 SSB Area Radio Resource Status List IE，排除 DL/UL scheduling PDCCH CCE usage IE，关闭相关 FFS。
- Cell To Report List IE 不用于请求 UE performance feedback；UE trajectory 与 UE performance 的比特移入 Node Measurement Initiation Result List IE。
- 放弃项：timing 相关新 cause value 无共识、Measurement Collection periodicity（留待 R19）、Data Collection Update 中的 UE performance feedback 上报终止指示——均 R18 不引入。
- 38.423 三份 TP（LB、UE Performance Data Collection、Failed Report Characteristics）获批定稿。

**ME（12.2.2.2）**
- Measured UE Trajectory 的 38.423 stage3 TP（CATT）获批，收敛 DATA COLLECTION REQUEST 中 Time Duration IE、Number of Visited Cells IE（1..16，与最大 16 小区一致）等编码细节。
- 放弃项：向请求节点显式指示一次性上报触发原因（Condition A/B/C/D，如 UE 进 IDLE/离开 target 节点/时长到期/切换次数达限）——R18 无共识，不引入。

**ES（12.2.2.3）**
- AI/ML Network Energy Saving 的 38.423 TP（Ericsson 等）获批，完成 Energy Cost (EC) 上报的 stage3 清理，并按 CATT 意见保留 "Energy Consumption is measured on a linear scale" 的表述。EC 维持节点级 INTEGER (0..10000, ...) 索引、由 OAM 配置归一化规则的既有结论。

**其他接口（12.2.2.4）**
- 本次无实质新共识（#121bis 已结论：split RAN 架构的 AI/ML 增强 R18 不做，NG/F1/E1 无新增标准影响）。

**Others（12.3）**
- 本次无实质新共识（continuous MDT 数据收集等增强此前已结论 R18 不再推进，本次无新文稿）。
