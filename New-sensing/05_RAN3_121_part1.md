# 3GPP R18 全周期 SON/AI agreed 议题调研（RAN3 #119–#124）

## RAN3#121 (2023.08.21–25, 图卢兹)【已详查】

### 本次会议 SON/MDT 主要共识与增强内容

**SHR/SPR**
- 细化 SPR 触发与优化归属：SN 发起经典 PSCell 变更时源 SN 定 T310/T312 门限、目标 SN 定 T304 门限；CPA 与 MN/SN 发起 CPC 由目标 SN 定 T304 并做根因分析；SN 内变更由源 SN 定 T310/T312 并做根因分析。
- 确定 SPR 转发机制：SPR 由"新节点"取回时一律先送回旧 MN，复用 ACCESS AND MOBILITY INDICATION（XnAP/F1AP）与 RAN Configuration Transfer（NGAP）转发；旧 MN 借助存储的 SN Mobility Information 定位旧源/目标 SN 的 UE 上下文。
- 系统间 SHR（NR→LTE）确定有用参数集（源/目标小区、测量结果、触发原因、UE 位置），SHR 转发采用"方案 3"（取回节点初步分析后转发给产生触发条件的节点）；LTE→NR 方向 T304 触发的 SHR 在不改 LTE 规范前提下支持（经 MobilityFromEUTRACommand 的 NR 容器下发配置）。以上经 R3-234716 LS 请 RAN2 确认。

**MRO**
- 语音回落：将 WA 升级为正式共识，在 S1/NG 系统间切换报告中定义新报告类型 "Inter-system mobility failure for voice fallback"（R3-234625 落入 38.413）。
- CPAC MRO：MN 收到 SCGFailureInformation 后做初步分析（区分 CPA/CPC、MN/SN 发起）；CPC 候选小区列表及执行条件纳入 MN→源 SN 消息（R3-234665）；CPAC 的 MRO 事件定义写入 37.340 新章节；UE 上报首个触发的 CPAC 事件及两次事件间时长。
- 快速 MCG 恢复：UE 至少上报 SCG 失败所在 PSCell、恢复失败原因（T316 超时/SCG 失败/SCG 去激活）及 SCG 失败类型。
- 为 CPAC 引入 UHI 记录的 Stage 2 描述（R3-234666）。

**RACH Enhancements**
- 确定网络侧 RACH 报告获取方案：F1 与 Xn 上均定义新的 Class 2 非 UE 关联消息 RACH INDICATION（DU→CU 指示其不可见的成功 RACH；SN→MN 指示 RA 报告可用），落入 38.420（R3-234742）、38.423（R3-234743）等。
- RA 报告增强候选方案收敛为 Opt2（RACH 分区配置信息）+ Opt3（接入到上报的时间），连同分析发 LS 请 RAN2 评估（R3-234643）。
- SN RA 报告无 X2/Xn 连接时的转发留给 gNB 实现。

**NPN**
- MDT SNPN 列表最大数目定为 16；不引入 SNPN-wide 区域范围；CAG 列表、SNPN Cell/TAI Based 区域范围等框架落入 TP（R3-234718/234719）与 37.320 基线 CR（R3-234720）；向 RAN2/SA5 发 LS 通报进展（R3-234744）。

**SON for NR-U**
- 在 XnAP 与 F1AP 的 RESOURCE STATUS UPDATE 中引入按 NR-U 信道粒度的可选负载指标（R3-234544/234545）；确认无需在 F1AP 中传递 UL EDT。

**MDT Enhancements**
- 本次无实质新共识（记录型 MDT 覆盖保护两篇提案均 noted，待 RAN2 进展）。

### 本次会议 AI/ML 主要共识与增强内容

**Stage 2 与通用流程**
- 标志性结论：新 Xn 过程族正式命名为 DATA COLLECTION REPORTING INITIATION（DATA COLLECTION REQUEST/RESPONSE/FAILURE）与 DATA COLLECTION REPORTING（DATA COLLECTION UPDATE）（R3-234724）。
- Requested Prediction Time 编码确定为 INTEGER、最大 60 秒、可扩展（R3-234663）；周期上报时预测时间点随上报周期顺延；R18 不引入时间区间；R18 不在 Xn 传递预测精度。
- Stage 2 文本更新（R3-234658）；过程设计原则强化：用例无关、数据类型无关（不指示数据用作输入/输出/反馈）、非 UE 关联为起点、R18 不做 NG 接口方案。

**负载均衡（LB）**
- 在 DATA COLLECTION REQUEST 中新增 Reporting Duration IE（UE 性能反馈上报时长，自切换执行成功起算）；支持周期性与一次性反馈；UE 转入 Idle/Inactive 或再次切换时终止反馈收集。
- 同意引入指示"定时类失败"的原因值；不引入"部分上报指示"与"请求信息组合失败"原因值。
- 相应 TP（R3-234689）因 Stage 3 细节分歧仅 noted。

**移动性优化（MO）**
- 实测 UE 轨迹上报确定为一次性上报，触发条件至少含：UE 状态切换、UE 离开目标节点、上报时长到期、同一目标节点内切换次数达配置值；Report time duration 与切换次数携带于 DATA COLLECTION REQUEST。
- Measured UE Trajectory 中小区 ID 列表必选，驻留时长存在性 FFS。

**网络节能（ES/NES）**
- Energy Cost 确定为节点级参数，R18 不做更细粒度；WA：EC 以线性刻度索引编码时由 OAM 配置归一化规则且区域内相邻节点一致。
- 同意把实测 EC 纳入 DATA COLLECTION 过程族（R3-234752：Report Characteristics 位图第 8 比特定义为 Energy Cost）。

**其他接口与面向 AI 的 MDT**
- E1/F1 是否需要改动分歧明显（SS/ZTE/CATT 支持，Nokia/E////HW 反对），未形成结论；面向 AI 的连续 MDT 讨论形成"信令型/管理型 MDT 能力差距"共同理解，LS 草稿均未发出。
