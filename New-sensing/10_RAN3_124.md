## RAN3#124 (2024.05.20–24, 福冈)

### 本次会议 SON/MDT 主要共识与增强内容

R18 纠错在第 9 章，SONMDT 专块（CB # 10_SONMDT，moderator Samsung），另有 9.4 Endorsed TEI CRs Review 中的转正项。

## RA report 的 PSCell List
- XnAP 纠错 CR1270（TS 38.423，Correction on PSCell List Container List of RA report）经 R3-243870/243942 修订后 agreed（R3-243967）。
- TS 37.340 的 PSCell List of RA report 纠错（R3-243871→R3-243943）endorsed；stage2 TS 38.401 CR0387 与 X2AP CR1788 因 EN-DC 场景细节争议（E/// 认为 X2AP CR 不必要）noted。

## MDT：CAG List ASN.1 纠错
- ASN.1 correction of CAG List for MDT：TS 38.413 CR1150（R3-243782）与 TS 38.423 CR1282（R3-243783）均 agreed，为 NBC（non-backward-compatible）ASN.1 修正。
- 9.4 节转正：user consent for trace reporting 的 TS 38.401 CR0384（澄清 AMF/UDM 均可触发 trace、修正 signalling based MDT activation 流程描述），WI code 更新为 R16 SONMDT 后 agreed（R3-243799）。

## NR-U 与 stage 2 补遗
- Stage 2 correction for NR-U（TS 38.401 CR0396）agreed（R3-243784），在 stage2 补齐 DL LBT failure 上报相关描述（stage3 方案上次会议未获同意）。
- Addition of SON for fast MCG recovery in stage 2（draftCR）：删除第二处改动后 endorsed（R3-243785），补上此前遗漏的 fast MCG recovery stage2 描述。
- Addition of CPAC detection in stage 2（draftCR）endorsed，补齐 CPAC MRO 检测的 stage2 描述。

## Inter-system MRO 与 SPR
- Inter-system MRO 纠错两份 draftCR（R3-243282→R3-243916、R3-243283→R3-243917）endorsed；其余 R16/R17 镜像 draftCR 未出结论，CB 继续讨论是否需要 inter-system MRO stage2 描述。
- SN-initiated flag signalling for SPR（Ericsson）被 CATT/Lenovo/HW 认定为 optimization 而非纠错，noted 不引入；SPR forwarding draftCR 因措辞问题 noted。

## 其他
- Correcting IE referenced in Coverage Modification（TS 38.423 CR1290 R17 / CR1291 R18）agreed，修正引用的 IE 错误。

### 本次会议 AI/ML 主要共识与增强内容

R18 AI/ML 纠错在第 9 章（CB # 14_AIMLforNGRAN，moderator Samsung），并有 9.4 Endorsed TEI CRs Review 转正项。议程 11 的 AI/ML SI（slicing、CCO 等）为 Rel-19，不在本摘要范围。

## AI/ML 术语定义与致 SA5 的 LS
- 解决 #123bis 遗留的术语问题：在 TS 38.401 的 3.1 Definitions 中新增 AI/ML Model Training 与 AI/ML Model Inference 定义，采用引用 SA5 TS 28.105 clause 3.1 中 "ML model training" 与 "AI/ML inference" 定义的方式；CR0397（Samsung/ZTE 等 → R3-243966）agreed。
- 配套 LS out "LS on terminology definitions for AI-ML in NG-RAN"（致 SA5，抄送 RAN1/RAN2/RAN）经多轮修订后 agreed（R3-243969）：请 SA5 注意 "ML model training" 与 "AI/ML Model Training" 的术语差异并反馈；由于 RAN WG 规范将引用 TS 28.105 的定义，若 SA5 修改被引用定义须通知相关 RAN WG。

## Stage 2 纠错
- Stage 2 corrections for AI/ML for NG-RAN（TS 38.300 draftCR，Huawei/Nokia 等 → R3-243928）endorsed；但 "model performance feedback" 表述的两个候选修改（删除 "out of 3GPP scope" 语句 vs 引用 TS 28.105 的 model performance indicators）最终 "No consensus"，该点未修。
- Stage 2 update on AI/ML for NG-RAN（CATT 等 draftCR → R3-243822）endorsed：新增/更名 "16.20.X Data Collection and Reporting" 小节，补齐遗漏的 stage2 描述。
- OAM requirements additions regarding AI/ML（Ericsson 等 draftCR）noted，继续等待 SA5 进展。
- UE performance metrics 定义讨论（per-UE 性能按 UE 各 DRB 测量均值导出）noted，未形成纠错。

## Stage 3 / Xn 纠错
- 9.4 节转正：Stage 3 corrections for AI/ML for NG-RAN（TS 38.423 CR1267r3，#123bis endorsed 版本）agreed。
- Correction on text description of UE trajectory information（TS 38.423 CR1262r2）agreed，即 #123bis 澄清的 Measured UE Trajectory / Predicted Trajectory Cell Information / Cell Based UE Trajectory Prediction IE 语义。
- Correction on Data Collection procedures over Xn（CR1316）：UE 数上限 16 还是 32 有争议，CR noted；其第一处修改交由 TS 38.423 rapporteur 作为 rapporteur's update 落实。
