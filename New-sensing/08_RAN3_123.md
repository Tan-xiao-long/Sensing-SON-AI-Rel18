## RAN3#123 (2024.02.26–03.01, 雅典；R18 纠错阶段)

### 本次会议 SON/MDT 主要共识与增强内容

议程 9.1.1（WID: NR_ENDC_SON_MDT_enh2-Core），含 Corrections 与 ASN.1 review 两部分。

## NPN 场景 MDT
- 收到 SA5 关于 MDT for NPN 的回复 LS（R3-240030，noted）；针对 SNPN Cell/TAI Based MDT 是否保留展开辩论（CB # SONMDT1_NPN），拟向 SA5 发的 [Draft] Reply LS（R3-241029）最终仅 noted，未发出。
- 修正 Area Scope of MDT 中 PNI-NPN Based MDT 下 PLMN ID IE 的 Presence 为 "M"，使 tabular 与 ASN.1 对齐（NGAP rapporteur 更新）。
- MDT NPN 的 ASN.1 纠错 CR 获批：TS 38.413 CR1086（R3-241119 agreed）、TS 38.423 CR1163（R3-241120 agreed）；TS 37.320 stage2 draftCR（R3-240712 相关的 R3-240711）endorsed。

## SHR/SPR 与 MRO
- 修正 TS 37.470 中 SPR optimization 描述（R3-241068 endorsed）；MRO 纠错（R3-241064 endorsed）；SN Mobility Information 的引用错误修正（R3-241065→R3-241116 agreed），即引用（reference）指向错误问题。
- SN-initiated flag for SPR 的 LS out 提案（R3-240760）noted，多数公司认为应等 RAN2 进展。
- Inter-system HO Report over NG 纠错 CR1077（TS 38.413）agreed。

## RACH 优化
- SN RACH report 纠错：TS 38.423 CR1229（R3-240870）与 TS 36.423 CR1780（R3-240871）agreed，多篇同类 CR 合并；同时核对 IE 命名。
- Stage 2 SON 描述补遗（Lenovo draftCR）：统一 EN-DC/NGEN-DC 为 MR-DC，补充 "UE may also support collection of SN RA Reports"、SN RA report 检索/转发在 TS 37.340 规定（R3-241123 endorsed）。
- F1AP RACH 纠错 CR1320（TS 38.473）：更新 Transaction ID IE 的 tabular/criticality（R3-241121 agreed）。

## NR-U（SON for NR-U）
- 澄清 DL LBT Failure Information Request/List IE 在 HANDOVER REQUEST 与 ACCESS AND MOBILITY INDICATION 中的处理行为，TS 38.423 CR1178（R3-240874）agreed。
- CB # SONMDT3_NRU 达成：在 UE CONTEXT SETUP/MODIFICATION REQUEST 中引入 DL LBT Failure Information Request IE，并新增 non-UE associated class 2 过程 DU-CU Access and Mobility Indication 用于 DU→CU 上报 DL LBT 失败次数；stage2 CR（R3-241084）与 F1AP stage3 CR（R3-241085→R3-241118）agreed。

## MRO for CPAC / fast MCG recovery
- TS 37.340 CPAC 纠错：补充 "CPC/CPA Execution to wrong PSCell" 场景描述（R3-240876 endorsed）；Suitable PSCell ID 是否引入未达共识；CPAC detection 与 fast MCG recovery 的 stage2 描述留待后续会议（To be continued）。

### 本次会议 AI/ML 主要共识与增强内容

议程 9.1.3（WID: NR_AIML_NGRAN-Core），含 Corrections 与 ASN.1 review。

## 与 SA5 的往来 LS
- SA5 来函通报 AI/ML Management 规范进展（R3-240032，noted）。RAN3 回复 LS（基于 R3-240447→R3-241183 agreed）：R18 阶段 RAN3 未涉及 AI/ML model 的 lifecycle management，仅聚焦支撑 AI/ML use case 所需数据的规范化，未识别出相关需求；仅限 R18 范围，不提 R19 SI。
- SA5 关于 Energy Cost index 的回复 LS（R3-240056）：RAN3 再回复 LS（Nokia 稿 R3-240733→R3-240998 agreed），澄清 Energy Consumption 最小/最大值并非单个 gNB 自身的极值，而是某区域内所有 gNB 的最小/最大能耗值；能耗值与 EC index 的映射规则由运营商实现决定（out of 3GPP scope）。

## Stage 3 纠错（TS 38.423）
- XnAP 综合纠错 CR1217（R3-241124）agreed，多家公司联署。
- "AI for RAN" 术语统一纠错（R3-241107→R3-241125）agreed。
- 两个遗留争议未获共识：新增 cause value "Measurement not supported with requested prediction time"（因时间/预测能力导致失败），以及在 Registration Request IE 中增加 "update/modify" code point 以修改 Data Collection 过程——均意见分裂，未通过。

## Stage 2 纠错
- Lenovo 术语纠错 draftCR（R3-240211→R3-241126 endorsed）：统一 "Support of AI/ML for NG-RAN is to facilitate AI/ML techniques in NG-RAN" 表述，并澄清 measured UE trajectory 与 UE performance feedback 的采集在 Handover 成功执行时触发。
- Ericsson/Nokia 提出的补充原则与定义（如部署前 testing 要求）因被认为过度限制或不必要而 noted。

## ASN.1 review（CB # AIML2_ASN.1）
- 修正 HANDOVER REQUEST 中 DATA COLLECTION ID IE 的 ASN.1。
- Node/Cell Measurement Failed Request Characteristics IE 的 bitmap 大小由 BITSTRING(SIZE(12832)) 之误修正为 32。
- 修正 Requested Prediction Time IE 的 ASN.1；DATA COLLECTION RESPONSE/UPDATE 消息中 cell ID 改为引用 GlobalNG-RANCell-ID。
- 澄清部分测量可用时的行为：NG-RAN node2 对 admitted 的请求发起上报，并在 DATA COLLECTION RESPONSE 中携带 Node/Per Cell Measurement Initiation Result List IE（XnAP rapporteur 更新）；杂项 ASN.1 清理 CR（R3-241086→R3-241128）agreed。
